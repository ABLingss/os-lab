### 实验1：Linux内核裁剪和编译 (4学时)

#### 需要配置的环境
- VMware虚拟机运行 Linux（Ubuntu/Debian/OpenEuler均可）
- 内核编译工具链：`gcc make binutils flex bison libncurses-dev libelf-dev libssl-dev bc`
- Linux内核源码（当前使用 linux-6.18.15，已下载到 `/usr/src/linux-6.18.15`）

#### 需要写的代码
- **无需写代码**，纯配置+编译操作

#### 需要执行的步骤
1. `sudo apt install` 安装编译依赖
2. 下载/拷贝内核源码到 `/usr/src`
3. `sudo cp /boot/config-$(uname -r) .config`
4. `sudo make olddefconfig`
5. `sudo make menuconfig` 进行内核裁剪
6. `sudo make -j$(nproc)` 编译
7. `sudo make modules_install`
8. `sudo make install`
9. `sudo update-grub`
10. `sudo reboot` 后用新内核启动
11. `uname -r` 验证

### 实验2：Linux系统调用分析及增加系统调用 (4学时)

#### 需要写的代码

**内核态（修改3个文件）：**
- `kernel/sys.c` — 添加 `sys_my_add()` 实现函数
- `arch/x86/include/uapi/asm/unistd_64.h` — 分配系统调用号 `#define __NR_my_add 450`
- `arch/x86/entry/syscall_64.c` — 在 `sys_call_table[]` 中添加映射

**用户态（test_mycall.c）：**
```c
#include <stdio.h>
#include <sys/syscall.h>
#define __NR_my_add 450
int main() {
    long result = syscall(__NR_my_add, 10, 20);
    printf("10 + 20 = %ld\n", result);
    return 0;
}
```
