## 实验概述

在 Linux 内核中实现**多级反馈队列调度算法（MLFQ）**，替代/补充 CFS 调度器。8 级队列，基础时间片 10 tick，每级翻倍，用完降级，周期性优先级提升防止饥饿。

- **学时**: 8
- **类型**: 内核编程（调度算法）
- **难度**: ⭐⭐⭐⭐⭐

**核心知识点**：
- Linux 进程调度框架（`struct sched_class`）
- CFS 完全公平调度器（`kernel/sched/fair.c`）
- 多级反馈队列（MLFQ）算法原理
- 时间片管理、优先级提升（priority boost）

## MLFQ 算法设计

```
         ┌─────────────────────────────────┐
         │     优先级提升（每100 tick）      │
         │   所有进程提升到Q0（防饥饿）      │
         └─────────────────────────────────┘
                        ↓
    ┌──────┐   ┌──────┐   ┌──────┐       ┌──────┐
    │  Q0  │ → │  Q1  │ → │  Q2  │ → ... │  Q7  │
    │10tick│   │20tick│   │40tick│       │1280t │
    └──┬───┘   └──┬───┘   └──┬───┘       └──────┘
       │          │          │               ↑
       └──────────┴──────────┴───────────────┘
              时间片用完 → 降级到下一级

调度规则：
  1. 始终从 Q0 开始扫描，选择第一个非空队列的队首任务
  2. 进程在分配给它的时间片内运行
  3. 时间片用完 → 降级到下一级队列（时间片翻倍）
  4. 每 100 tick 将所有进程提升到 Q0（防饿死）
```

### 队列参数

| 队列 | 时间片 (tick) | 说明 |
|------|-------------|------|
| Q0 | 10 | 最高优先级，短时间片 → I/O密集型快速响应 |
| Q1 | 20 | 次高优先级 |
| Q2 | 40 | |
| Q3 | 80 | |
| Q4 | 160 | |
| Q5 | 320 | |
| Q6 | 640 | |
| Q7 | 1280 | 最低优先级，长时间片 → CPU密集型批量运行 |

## 步骤

### 1. 修改内核调度代码

```bash
cd /usr/src/linux-6.18.15
sudo vim kernel/sched/core.c
```

在 `__schedule()` 函数的 `pick_next_task()` 调用之后添加：

```c
if (next && next->pid != 0) {
    printk(KERN_INFO "[MLFQ] Switched to %s (PID=%d, prio=%d, policy=%d)\n",
           next->comm, next->pid, next->prio, next->policy);
}
```

同时可以添加 MLFQ 级别模拟变量和降级/提升逻辑（参见 `exp11_mlfq_ref.c`）。

## 实验原理要点（供写报告参考）

### CFS vs MLFQ
- **CFS** (完全公平调度)：按 vruntime（虚拟运行时间）公平分配 CPU，红黑树组织
- **MLFQ** (多级反馈队列)：多级队列 + 动态优先级调整，兼顾响应时间和吞吐量

### MLFQ 三条规则
1. **优先级规则**：A 优先级 > B 优先级 → A 先运行
2. **时间片规则**：同优先级时间片轮转，用完降级
3. **提升规则**：周期性将所有进程提升到最高优先级（防饥饿）

### 为什么 MLFQ 有效
- I/O 密集型任务在 Q0（短时间片）快速响应
- CPU 密集型任务逐渐降级到低优先级长队列
- 优先级提升确保低优先级任务不会永远饥饿
