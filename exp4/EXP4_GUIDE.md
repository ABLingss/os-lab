# 实验4：Linux下生产者/消费者问题的多进程实现 — 操作指导

> 严格按指导书「实验四」内容编写 | 用户态编程 (C/XSI IPC)

## 实验概述

使用 XSI 信号量集 + 共享内存实现经典的**生产者-消费者问题**。通过 `fork()` 创建 3 个生产者进程和 4 个消费者进程，共享内存作为有界缓冲区，信号量集实现互斥与同步。

- **学时**: 4
- **类型**: 用户态 C 编程
- **难度**: ⭐⭐⭐

**核心知识点**：
- XSI 信号量集 (`semget`, `semctl`, `semop`) — 互斥 + 同步
- 共享内存 (`shmget`, `shmat`, `shmdt`, `shmctl`) — 进程间数据共享
- `fork()` + `wait()` — 多进程编程

代码文件：`exp4_producer_consumer.c`

### 系统架构

```
                    ┌──────────────┐
                    │   共享内存     │
                    │  (环形缓冲区)  │
                    │  buffer[8]    │
                    └──┬──┬──┬──┬──┘
                       │  │  │  │
              ┌────────┘  │  │  └────────┐
              │           │  │           │
         ┌────▼──┐   ┌───▼──▼───┐   ┌───▼──────┐
         │ MUTEX │   │  EMPTY   │   │   FULL    │
         │  = 1  │   │   = 8    │   │    = 0    │
         └───────┘   └──────────┘   └──────────┘
         (互斥信号量)  (空闲槽位)     (已占用槽位)

   生产者1  生产者2  生产者3     消费者1 消费者2 消费者3 消费者4
    (fork)   (fork)   (fork)      (fork)  (fork)  (fork)  (fork)
```

### 信号量集设计

| 信号量 | 索引 | 初值 | 含义 |
|--------|------|------|------|
| MUTEX | 0 | 1 | 互斥访问共享缓冲区 |
| EMPTY | 1 | BUFFER_SIZE(8) | 空闲缓冲区槽位数 |
| FULL | 2 | 0 | 已占用缓冲区槽位数 |

### 生产者流程

```
while (还有数据要生产) {
    生产一个数据项;
    P(EMPTY);    // 等待空闲槽位
    P(MUTEX);    // 进入临界区
    将数据放入缓冲区;
    V(MUTEX);    // 离开临界区
    V(FULL);     // 增加已占用槽位
}
```

### 消费者流程

```
while (还有数据要消费) {
    P(FULL);     // 等待有数据
    P(MUTEX);    // 进入临界区
    从缓冲区取出数据;
    V(MUTEX);    // 离开临界区
    V(EMPTY);    // 增加空闲槽位
    消费数据项;
}
```

## 实验原理要点（供写报告参考）

### 生产者-消费者问题
生产者进程产生数据放入缓冲区，消费者进程从缓冲区取出数据。关键约束：
- 缓冲区满时生产者不能放入（需等待消费者取走）
- 缓冲区空时消费者不能取出（需等待生产者放入）
- 多个生产者/消费者需互斥访问缓冲区

### XSI 信号量集 vs POSIX 信号量
- XSI 信号量集（`semget/semop`）：内核级，可同时操作多个信号量（原子操作集），适合本实验
- 信号量初值通过 `semctl(SETVAL)` 设置
