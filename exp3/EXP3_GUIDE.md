# 实验3：Linux下哲学家进餐问题的多线程实现与解决 — 操作指导

## 实验概述

使用 pthread 多线程库实现**哲学家进餐问题**，分别演示死锁发生和死锁预防两种场景。

- **学时**: 4
- **类型**: 用户态 C 编程
- **难度**: ⭐⭐

**核心知识点**：
- 死锁的四个必要条件（互斥、持有并等待、不可抢占、循环等待）
- 使用 `pthread_mutex_lock` / `pthread_mutex_unlock` 实现互斥
- 通过**破坏循环等待条件**预防死锁（奇偶号哲学家拿筷子顺序不同）

---

## 代码说明

代码文件：`exp3_philosophers.c`

### 程序结构

| 组件 | 说明 |
|------|------|
| `philosopher_deadlock()` | 死锁版本：所有哲学家先拿左筷子再拿右筷子 |
| `philosopher_prevent()` | 预防版本：奇数号先左后右，偶数号先右后左 |
| `deadlock_watcher()` | 监视线程：检测哲学家是否饥饿超时（判定死锁） |

### 两种模式

1. **死锁版本** — 所有哲学家统一先拿左筷子，再拿右筷子。当5人同时拿起左筷子等待右筷子时，形成循环等待死锁。
2. **预防版本** — 奇数号哲学家先左后右，偶数号先右后左。破坏循环等待条件，死锁无法发生。

## 实验原理要点（供写报告参考）

### 死锁的四个必要条件
1. **互斥条件**: 每根筷子同时只能被一位哲学家持有
2. **持有并等待**: 哲学家持有一根筷子等待另一根
3. **不可抢占**: 筷子不能被强行夺走
4. **循环等待**: 每位哲学家都在等待右边邻居释放筷子，形成环

### 死锁预防策略
本实验采用**破坏循环等待条件**：给筷子编号 0~4，规定奇数号哲学家先拿左筷再拿右筷，偶数号先拿右筷再拿左筷。这样哲学家0（偶数）先拿筷4再拿筷0，哲学家1（奇数）先拿筷1再拿筷2，打破了可能的循环等待环。
